# Задания

## Задание 1. Ранжирование продуктов по цене

Задача:
Ранжировать продукты (**products**) по цене (**base_msrp**) от дорогих к дешёвым **внутри каждого типа продукта (product_type)**.

Для решения используется оконная функция **RANK()** с разделением данных по типу продукта.

### SQL-запрос

```sql
SELECT 
    product_id,
    product_type,
    base_msrp,
    RANK() OVER (
        PARTITION BY product_type
        ORDER BY base_msrp DESC
    ) AS price_rank
FROM products;
```

### Объяснение

`PARTITION BY product_type`
разделяет все продукты на группы по типу.

(Простыми словами: **каждый тип продукта считается отдельно**.)

`ORDER BY base_msrp DESC`
сортирует продукты внутри типа по цене **от дорогих к дешёвым**.

`RANK()`
присваивает ранг каждому продукту.

(Простыми словами: **самый дорогой товар получает 1 место**.)



# Задание 2. Сравнение дат регистрации клиентов

Задача:
Для каждого клиента вывести:

* дату регистрации
* дату регистрации **предыдущего клиента в том же штате**

Для решения используется функция **LAG()**.

### SQL-запрос

```sql
SELECT 
    customer_id,
    state,
    date_added,
    LAG(date_added) OVER (
        PARTITION BY state
        ORDER BY date_added
    ) AS previous_customer_date
FROM customers;
```

---

### Объяснение

`PARTITION BY state`
делит клиентов по штатам.

(Простыми словами: **сравнение происходит только между клиентами одного штата**.)

`ORDER BY date_added`
упорядочивает клиентов по дате регистрации.

`LAG(date_added)`
возвращает дату регистрации **предыдущего клиента**.

(Простыми словами: **показывает кто зарегистрировался перед текущим пользователем**.)

# Задание 3. Нарастающий итог продаж

Задача:
Рассчитать **нарастающий итог продаж (sales_amount)** по дням для дилера:

```
dealership_id = 10
```

Для этого используется агрегатная функция **SUM()** в оконном режиме.

---

### SQL-запрос

```sql
SELECT 
    sales_transaction_date::DATE AS sale_date,
    sales_amount,
    SUM(sales_amount) OVER (
        ORDER BY sales_transaction_date
    ) AS running_total
FROM sales
WHERE dealership_id = 10
ORDER BY sales_transaction_date;
```

---

### Объяснение

`WHERE dealership_id = 10`
выбирает продажи только одного дилерского центра.

`ORDER BY sales_transaction_date`
сортирует продажи по датам.

`SUM(sales_amount) OVER (...)`
считает **накопительную сумму продаж**.
