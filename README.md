# SQL-2026
# Лабораторные работы по SQL

### [Лабораторная работа №1: Основы SQL (SELECT, WHERE, ORDER BY, CRUD)](https://github.com/BratchenkoAA/SQL-2026/tree/main/SQL-lb/SQL-lb_01)
В ходе первой работы были освоены базовые операции SQL: выборка данных с фильтрацией (`WHERE`), сортировка (`ORDER BY`), а также полный цикл CRUD-операций — создание таблицы (`CREATE TABLE AS`), обновление (`UPDATE`) и удаление записей (`DELETE`). Получены навыки работы с условиями `BETWEEN` и базовой структурой SQL-запросов.


### [Лабораторная работа №2: Соединения таблиц и преобразование данных](https://github.com/BratchenkoAA/SQL-2026/tree/main/SQL-lb/SQL-lb_02)

Вторая работа была посвящена работе с несколькими таблицами: изучены различные типы соединений (`INNER JOIN`, `LEFT JOIN`), применение подзапросов (`Subquery`) для решения сложных задач поиска, а также использование условной логики с конструкцией `CASE WHEN` для классификации данных и создания вычисляемых столбцов.

### [Лабораторная работа №3: Агрегация данных](https://github.com/BratchenkoAA/SQL-2026/tree/main/SQL-lb/SQL-lb_03)
В третьей работе освоены методы агрегирования данных: применение агрегатных функций (`MIN`, `MAX`, `AVG`, `COUNT`, `SUM`), группировка данных с помощью `GROUP BY`, а также фильтрация сгруппированных результатов через `HAVING`. Полученные навыки позволяют проводить статистический анализ и формировать сводные отчеты по данным.


### [Лабораторная работа №4:Использование оконных функций в SQL](https://github.com/BratchenkoAA/SQL-2026/blob/main/SQL-lb/lb_4%20(2).md)
В четвёртой работе изучены оконные функции SQL: ранжирование записей с помощью RANK(), сравнение строк через LAG/LEAD и расчёт накопительных итогов через SUM() OVER(). Полученные навыки позволяют выполнять сложный аналитический анализ данных (рейтинги, динамика, скользящие суммы) без потери исходной детализации строк.

### [Лабораторная работа №5: Оптимизация запросов с помощью индексов](https://github.com/BratchenkoAA/SQL-2026/blob/main/lb_5.md)
В пятой работе изучены методы оптимизации SQL-запросов: анализ плана выполнения через EXPLAIN ANALYZE, сравнение последовательного сканирования (Seq Scan) и сканирования по индексу (Index Scan), а также создание B-Tree индексов для ускорения фильтрации (WHERE) и диапазонных условий (BETWEEN). Полученные навыки позволяют диагностировать "медленные" запросы, принимать обоснованные решения о создании индексов и значительно повышать производительность базы данных без изменения самого SQL-кода.
# ClickHouse
###  [ПЗ №1. Основы ClickHouse: установка, типы данных, движки таблиц](https://github.com/BratchenkoAA/SQL-2026/tree/main/SQL_CH_01)
В этой работе освоены методы работы с колоночной СУБД ClickHouse: создание таблиц с движками семейства `MergeTree` (`MergeTree`, `ReplacingMergeTree`, `SummingMergeTree`), партиционирование данных по месяцам, использование оптимизированных типов данных (`DateTime64`, `Decimal`), а также выполнение аналитических запросов с агрегацией, фильтрацией по партициям и объединением таблиц. Полученные навыки позволяют эффективно хранить, обновлять и агрегировать большие объёмы данных для задач аналитики и построения отчётности.
